# 技术规范应用示例

> 本示例展示如何在实际项目中应用 Vibe Coding Team 的技术规范。

---

## 示例项目: 任务管理系统 (Todo App)

### 项目概述

一个简单的任务管理系统，展示：
- 设计系统的应用
- API 规范的实施
- 代码规范的遵循
- 安全规范的落实
- 测试规范的实践

---

## 示例内容

### 1. 设计系统应用

**文件:** `design-system-example.md`

展示如何在 UI 组件中应用设计系统规范：
- 色彩系统的使用
- 间距系统的应用
- 字体规范的遵循
- 组件设计的标准

### 2. API 规范实施

**文件:** `api-example.md`

展示 RESTful API 的设计：
- URL 设计规范
- 请求/响应格式
- 错误处理标准
- 状态码使用

### 3. 代码规范示例

**文件:** `code-standards-example.md`

展示代码组织：
- 命名规范
- 代码风格
- 注释规范
- 最佳实践

### 4. 数据库设计

**文件:** `database-example.md`

展示数据库规范：
- 表结构设计
- 索引设计
- 命名规范

### 5. 安全实现

**文件:** `security-example.md`

展示安全规范：
- 密码存储
- JWT 实现
- 输入验证
- 访问控制

### 6. 测试实践

**文件:** `testing-example.md`

展示测试规范：
- 单元测试
- 集成测试
- TDD 实践

---

## 快速开始

```bash
# 查看设计系统示例
cat design-system-example.md

# 查看 API 规范示例
cat api-example.md

# 查看代码规范示例
cat code-standards-example.md
```

---

## 技术栈

- **前端**: React + TypeScript + Tailwind CSS
- **后端**: Node.js + Express + TypeScript
- **数据库**: PostgreSQL
- **测试**: Jest + React Testing Library + Supertest
